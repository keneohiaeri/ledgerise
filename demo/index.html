Schema & Types

Operator-facing reference for the canonical transaction record. These fields are produced by inbound adapters and consumed by the journal engine.

1
Raw source event Webhook, poll result, file row, or manual entry
→
2
Inbound adapter Validates, masks PII, converts amounts, normalizes fields
→
3
Canonical record Single stable format for mapping and journal generation
Posting Rules
Posted Only settled records with settled_at can become journal entries.
Blocked failed, pending, and disputed records are stored but not posted.
Review No matching mapping rule means suspense/unmapped, never silent dropping.
Reversal reversed records mirror the original posted journal when reversal_of is present.
Mapping Priority
  1. product.line + product.biller
  2. product.line + product.biller_category
  3. product.line catch-all
  4. Suspense account if no rule matches

Adapter Modes
Webhook

Source system pushes each event to Ledgerise. Best for payment processors that support reliable event callbacks.

paystack-webhook
Poll

Ledgerise calls the source API on a schedule using a cursor such as last fetched time or source ID. Best when the provider does not push events.

mpesa-poll
File Import

Operator uploads CSV/XLSX exports. Useful for onboarding, backfills, banks, and providers without an API.

csv-import
Manual Entry

Structured one-off entry for exceptions. Stored through the same canonical schema but not a replacement for automation.

manual-entry
